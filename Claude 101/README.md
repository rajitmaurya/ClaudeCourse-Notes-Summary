## 1. Course Overview

Claude 101 is a beginner-friendly introduction to working with Claude — Anthropic's AI assistant. The course is designed for anyone who wants to start using Claude effectively, whether for personal productivity, professional work, or creative projects. No prior AI experience is required.

Over 14 lessons, you will go from understanding what Claude is, to having your first conversation, to using advanced features like Projects, Artifacts, Skills, and enterprise-level tools. By the end, you will have a solid foundation to integrate Claude into your daily workflow confidently.

## 2. Meet Claude

This section introduces you to Claude — what it is, how it thinks, and how to start talking to it effectively.

### 2.1 What is Claude?

Claude is an AI assistant made by Anthropic, designed to be helpful, harmless, and honest. Unlike a search engine that returns links, Claude holds a conversation with you — it reads what you write, understands the context, and responds with thoughtful, relevant answers.

Key things to know about Claude:

- **It is a large language model (LLM):** Claude is trained on vast amounts of text and has broad knowledge across topics including writing, coding, analysis, math, science, history, and more.
- **It is conversational:** You interact with Claude through natural language — just write as you would to a colleague or assistant.
- **It is not the internet:** Claude has a knowledge cutoff date and does not browse the web by default (unless given tools to do so). It works from what it learned during training and what you tell it in the conversation.
- **It is designed to be safe:** Anthropic builds Claude with safety and ethics as a core priority. Claude will decline requests that could cause harm and aims to be transparent about its limitations.
- **It has context memory within a session:** Claude remembers everything said earlier in the same conversation, allowing it to build on prior exchanges — but it does not remember past conversations by default unless Projects are used.

### 2.2 Your First Conversation with Claude

This lesson walks you through opening Claude and having your very first interaction. Topics covered:

- **Accessing Claude:** Via claude.ai in a browser, or via the desktop app.
- **The chat interface:** Understanding the input box, conversation thread, and response area.
- **Writing your first prompt:** How to phrase a simple question or request.
- **Reading Claude's response:** What a typical response looks like and how to interpret it.
- **Following up:** How to ask a follow-up question, ask for clarification, or request a different approach.
- **Regenerating responses:** How to ask Claude to try again if the first answer is not what you needed.

The goal of this lesson is to demystify the experience — talking to Claude should feel natural, like messaging a knowledgeable friend.

### 2.3 Getting Better Results

The quality of Claude's responses depends heavily on how you prompt it. This lesson teaches the fundamentals of effective prompting:

- **Be specific:** Vague prompts get vague answers. The more context and detail you provide, the better Claude can help.
- **State your goal:** Tell Claude what you are trying to accomplish, not just what you want it to do. For example: "I'm writing a cover letter for a software engineering role — can you help me make this paragraph more compelling?"
- **Set the format:** If you need a bullet list, a table, a short paragraph, or a specific word count, say so explicitly.
- **Assign a role:** Asking Claude to act as an expert (e.g., "You are an experienced financial analyst") can sharpen the tone and depth of its responses.
- **Iterate:** Treat prompting as a conversation. If the first response is close but not quite right, tell Claude what to adjust rather than starting over.
- **Use examples:** Showing Claude an example of what you want (a sample paragraph, a data format) is one of the fastest ways to get the right output.

## 3. Claude Desktop App: Chat, Cowork, Code

This lesson introduces the Claude desktop application — a dedicated app for Mac and Windows that goes beyond the browser experience. The desktop app has three core modes:

### Chat
The standard conversational interface. Use it for questions, writing, brainstorming, summarizing documents, and everyday tasks. The chat experience in the desktop app is the same as claude.ai but with the convenience of a native application — faster to open, easier to keep running in the background, and accessible via keyboard shortcuts.

### Cowork
Cowork mode is designed for longer, more structured work sessions where you and Claude collaborate on a document or task over time. Instead of a back-and-forth chat thread, Cowork gives you a workspace where Claude's outputs appear in an editable panel alongside the conversation. This is ideal for:

- Drafting and refining long documents (reports, proposals, articles).
- Working through complex problems step by step.
- Keeping your instructions and Claude's outputs organized in one place.

### Code
Code mode optimizes the interface for software development tasks. It provides:

- Syntax-highlighted code blocks.
- Easy copy-paste for code snippets.
- A workflow suited for debugging, code review, writing functions, and explaining technical concepts.
- Integration with your development environment (where supported).

The desktop app brings all three modes together in a single, fast, native experience that makes Claude easier to use throughout your workday.

## 4. Organizing Your Work and Knowledge

As you use Claude more, you will want ways to keep things organized and reusable. This section covers Projects, Artifacts, and Skills — three features that turn Claude from a one-off chatbot into a persistent work tool.

### 4.1 Introduction to Projects

Projects are a way to give Claude persistent context across multiple conversations. Instead of re-explaining your background, preferences, or project details every time you start a new chat, you can store that information in a Project and Claude will always have it available.

**What a Project contains:**
- **Instructions:** A set of guidelines Claude follows in every conversation within the project (e.g., "Always respond in formal English" or "This project is for a fintech startup targeting small business owners").
- **Knowledge files:** Documents, notes, or reference material you upload so Claude can draw on them in any conversation (e.g., your company's style guide, a product spec, a dataset).
- **Conversation history:** All chats within a project are saved and organized together.

**When to use Projects:**
- Ongoing work with a consistent context (a specific client, a recurring report, a long-term research topic).
- Team use cases where shared instructions and files ensure consistency across users.
- Any situation where you find yourself re-pasting the same context at the start of every chat.

### 4.2 Creating with Artifacts

Artifacts are standalone pieces of content that Claude creates and presents in a dedicated panel alongside the conversation. Unlike a regular chat response (which is part of the thread), an Artifact is a discrete, editable output.

**Types of Artifacts Claude can create:**
- **Documents:** Formatted text like reports, essays, emails, and proposals.
- **Code:** Scripts, functions, full applications, or components in any programming language.
- **Data visualizations:** Charts and graphs rendered from data you provide.
- **HTML/React components:** Interactive web elements that render live in the Artifact panel.
- **Markdown files:** Structured reference documents, READMEs, wikis.

**Why Artifacts are useful:**
- You can edit the Artifact directly without re-prompting Claude from scratch.
- Artifacts can be downloaded, copied, or shared easily.
- They keep the output cleanly separated from the conversation, so the chat thread stays readable.
- Claude can iterate on an Artifact across multiple turns — refining, expanding, or restructuring it based on your feedback.

### 4.3 Working with Skills

Skills are pre-built capabilities or templates that extend what Claude can do within a specific domain. They act as specialized tools you can activate for particular types of tasks.

**Examples of Skills:**
- A writing skill that applies a specific tone, structure, or style.
- A data analysis skill that knows how to interpret and summarize spreadsheets.
- A coding skill tuned for a particular language or framework.
- A research skill that follows a structured methodology for gathering and synthesizing information.

**How Skills work:**
- Skills can be provided by Anthropic, your organization, or (in some configurations) built by you.
- Activating a skill changes how Claude approaches a task — it applies the right structure, tone, and methodology automatically.
- Skills reduce the need for lengthy prompt engineering by encoding best practices into the tool itself.

## 5. Expanding Claude's Reach

Claude becomes dramatically more powerful when connected to external tools and data sources. This section covers how to extend Claude beyond the conversation window.

### 5.1 Connecting Your Tools

Claude can integrate with third-party tools and services via the **Model Context Protocol (MCP)** and built-in integrations. This allows Claude to take actions and retrieve information from the apps you already use.

**Examples of tool integrations:**
- **Google Drive / Dropbox:** Claude can read documents stored in your cloud drive and reference them in conversation.
- **Notion / Confluence:** Claude can pull in pages, notes, and wikis as context.
- **Jira / Asana:** Claude can look up tasks, projects, and statuses.
- **Slack:** Claude can search message history or draft messages.
- **GitHub:** Claude can read code, review pull requests, or look up issues.
- **Databases:** Claude can query structured data sources and return results in natural language.

**Why this matters:**
Connecting tools means Claude is no longer limited to what you paste into the chat. It can go and get the information it needs, act on your behalf in other systems, and give you answers grounded in your real, live data — not just its training knowledge.

### 5.2 Enterprise Search

Enterprise Search is a feature designed for organizations that want Claude to search across their internal knowledge base — documents, wikis, emails, tickets, and more — to answer employee questions.

**How it works:**
- Your organization's internal content is indexed and made searchable by Claude.
- When an employee asks Claude a question, Claude can search across that content and surface the most relevant answer, citing the source document.
- Claude synthesizes information from multiple sources rather than returning a list of links.

**Use cases:**
- "What is our policy on remote work expenses?" → Claude searches the HR handbook and returns the answer.
- "What did we decide in the Q2 planning meeting?" → Claude searches meeting notes and summarizes the decision.
- "Who is the technical lead for the payments team?" → Claude searches the org chart or team directory.

Enterprise Search turns Claude into an intelligent knowledge assistant for your entire organization.

### 5.3 Research Mode for Deep Dives

Research Mode is a specialized Claude capability for in-depth, multi-step research tasks. Instead of returning a single response, Research Mode has Claude:

1. **Plan the research:** Break the question into sub-questions and identify what needs to be found.
2. **Search broadly:** Use web search and connected sources to gather information from multiple angles.
3. **Synthesize findings:** Combine what it finds into a coherent, well-structured report.
4. **Cite sources:** Include references so you can verify the information and read further.

**When to use Research Mode:**
- Competitive analysis (e.g., "Give me a detailed breakdown of our top 5 competitors' pricing models").
- Market research (e.g., "What are the current trends in sustainable packaging for consumer goods?").
- Technical deep dives (e.g., "Explain the tradeoffs between different vector database architectures").
- Academic or policy research requiring synthesis of multiple sources.

Research Mode is significantly more thorough than a standard chat response and is designed for tasks where quality and depth matter more than speed.

## 6. Putting It All Together

### 6.1 Claude in Action: Use-Cases by Role

This lesson demonstrates how professionals in different roles use Claude in their day-to-day work. It shows that Claude is not a single-purpose tool — its value shifts depending on who you are and what you do.

**For Executives & Managers:**
- Summarizing long reports and documents into executive briefs.
- Drafting communications, announcements, and strategic memos.
- Preparing for meetings by generating briefing documents from raw notes.
- Analyzing options and getting structured pros/cons for decisions.

**For Writers & Marketers:**
- Drafting blog posts, social media content, ad copy, and email campaigns.
- Editing and proofreading for tone, clarity, and grammar.
- Brainstorming campaign ideas and content calendars.
- Repurposing content across formats (e.g., turning a webinar into a blog post).

**For Analysts & Researchers:**
- Summarizing research papers and reports.
- Cleaning, interpreting, and describing datasets.
- Generating hypotheses and research frameworks.
- Building structured literature reviews.

**For Developers & Engineers:**
- Writing, reviewing, and debugging code.
- Explaining unfamiliar codebases or APIs.
- Generating unit tests and documentation.
- Prototyping scripts and automation workflows.

**For Customer-Facing Teams:**
- Drafting responses to customer queries at scale.
- Summarizing customer feedback themes.
- Creating FAQs and support documentation.
- Role-playing difficult customer scenarios for training.

**For HR & Operations:**
- Writing job descriptions, onboarding materials, and policy documents.
- Summarizing employee survey results.
- Drafting performance review frameworks.
- Answering internal policy questions via Enterprise Search.

### 6.2 Other Ways to Work with Claude

Beyond the main chat interface and desktop app, there are several other ways to access and integrate Claude:

- **Claude API:** Developers can access Claude directly via Anthropic's API to build Claude-powered applications, internal tools, and automated workflows. The API supports all of Claude's capabilities including tool use, document processing, and streaming.
- **Claude in Chrome:** A browser extension that brings Claude into your browsing experience — summarize pages, ask questions about what you are reading, and get assistance without switching tabs.
- **Claude in Excel:** An integration that lets you use Claude directly within Microsoft Excel for tasks like formula writing, data cleaning, column generation, and natural language data analysis.
- **Claude in PowerPoint:** Bring Claude into your slide creation workflow — generate slide content, rewrite copy, and structure presentations from within PowerPoint.
- **Third-party integrations:** Claude is available through a growing ecosystem of third-party platforms and tools, from project management software to customer support platforms to coding environments.
